9. Help Users Recognize, Diagnose, and Recover from Errors

"Error messages should be expressed in plain language (no codes), precisely indicate the problem, and constructively suggest a solution."

When a user searches in the directory but there are no matching results, they are presented with an error message that suggests a few ways to find what they needed. The suggestions are helpful, however it might be useful to provide the user with items that are similar in spelling as well.That way there’s a chance that a simple typo won’t force the user to begin their search from scratch again.



When going to the page called “Paws,” and you click, “having problems logging in?,” it opens a separate window that doesn’t really help much except that it tells you which browsers to use. When clicking a link inside the pop-up box called “pop-up browsers,” it just leads you to the same information, resulting in useless and unnecessary links inside the pop-up box. Also, when clicking on the “Login” button with false information, it just leads you to a different page that says “Failed Login” and it does have a link to go back to the Login page, but I thought that this extra page just to say “Failed Login” was a waste of a page, since it doesn’t really offer any solutions to help out except to try again with a different login or password.



Also, with this Login page, there is no other link to go back to the homepage of GSU, just a logo at the top that seems like it’ll link you back to the homepage, but it does not. It actually has a link to the homepage in small letters in the description to Paws, “PAWS is Georgia State University’s portal...” which isn’t continuous with their web design and instead of making the logo at the header linkable, they made the homepage link in small letters in descriptions, which is highly inconsistent.
